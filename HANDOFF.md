## Apps Script プロジェクト

ローカルの `.clasp.json` に scriptId を保存済み（Git 管理外）。

エディタを開く:

```bash
cd gas-billing-check-app
npm run clasp:open
```

> **clasp 3.x の注意:** 旧コマンド `clasp open` は使えません。`npm run clasp:open`（内部で `clasp open-script`）を使ってください。

ブラウザで直接開く場合は `.clasp.json` の `scriptId` を使います。

```text
https://script.google.com/d/【scriptId】/edit
```

## 初回セットアップ手順（詳細）

全体の流れは次の3段階です。

1. 既存マスタのスプレッドシート ID を控える
2. Apps Script エディタで `runInitialSetup` を1回実行する
3. Webアプリとしてデプロイする

---

### ステップ0: 前提の確認

次がそろっていることを確認してください。

- Google アカウントでログインできる
- **基本情報マスタ**がある既存スプレッドシートにアクセスできる
  - シート名: `基本情報マスタ`
  - シート名: `利用終了者マスタ`
- ローカルに `gas-billing-check-app` フォルダがあり、clasp で GAS プロジェクトと紐付け済み
  - 紐付け済みなら `.clasp.json` が存在する（Git 管理外）

---

### ステップ1: マスタのスプレッドシート ID を確認する

他アプリ（基本情報アプリ等）で使っているマスタスプレッドシートと**同じもの**を指定します。

**ID の見つけ方**

1. 基本情報マスタがあるスプレッドシートをブラウザで開く
2. アドレスバーの URL を見る

```text
https://docs.google.com/spreadsheets/d/【ここがスプレッドシートID】/edit
```

3. `/d/` と `/edit` の間の長い文字列をコピーする

例:

```text
1mE1TsCiWdDkUoVLn7gBcI73TV03jQogqb_WwSD7kaFNb4HTwxHtK5wHR
```

**別の確認方法（他アプリを使っている場合）**

- 基本情報アプリ等の GAS エディタ →「プロジェクトの設定」→「スクリプト プロパティ」
- `MASTER_SPREADSHEET_ID` の値が同じ ID です

---

### ステップ2: Apps Script エディタを開く

ターミナルで次を実行します。

```bash
cd gas-billing-check-app
npm run clasp:open
```

ブラウザで Apps Script エディタが開きます。  
左側に `Code.gs` や `Setup.gs` などのファイルが並んでいれば OK です。

---

### ステップ3: `runInitialSetup` を実行する

これが初回セットアップの本体です。手動でスプレッドシートを作ったり、Script Properties を入力したりする必要はありません。

**実行手順**

1. エディタ上部の関数プルダウンから **`runInitialSetup`** を選ぶ
2. その右の「実行」ボタン（▶）を押す
3. 初回は「承認が必要です」と出るので、画面の指示に従って権限を許可する
   - スプレッドシートの作成・読み書き
   - 外部スプレッドシート（マスタ）の読み取り
   などが求められます
4. 再度「実行」を押す
5. 実行ログに次のような結果が出れば成功です

```json
{
  "message": "初期セットアップが完了しました。",
  "billingSpreadsheetId": "新しく作られた請求用SSのID",
  "billingSpreadsheetUrl": "https://docs.google.com/spreadsheets/d/...",
  "masterSpreadsheetId": "指定したマスタのID",
  "version": "0.1.1"
}
```

**実行するコード（エディタ下部の実行タブで直接入力する場合）**

関数プルダウンで引数を渡せない場合は、一時的に次のように書いて実行しても構いません。

```js
function runInitialSetupOnce() {
  // ここを実際のマスタ ID に書き換える
  var result = runInitialSetup('1mE1TsCiWdDkUoVLn7gBcI73TV03jQogqb_WwSD7kaFNb4HTwxHtK5wHR');
  Logger.log(result);
}
```

1. 上記を `Code.gs` の末尾などに貼り付け、ID を書き換える
2. 関数プルダウンで `runInitialSetupOnce` を選んで実行
3. 成功したら、貼り付けた一時関数は削除してよい

**この関数が自動で行うこと**

| 処理 | 内容 |
|---|---|
| 請求用スプレッドシート作成 | 名前「請求確定チェック」の新規スプレッドシート |
| Script Properties 登録 | `BILLING_SPREADSHEET_ID` と `MASTER_SPREADSHEET_ID` |
| シート作成 | 下記9シートの見出し行を自動作成 |
| 設定シート更新 | `MASTER_SPREADSHEET_ID` を設定シートにも記録 |

**作成されるシート一覧**

| シート名 | 用途 |
|---|---|
| 請求月管理 | 対象月ごとの状態・集計 |
| ほのぼの取込 | ほのぼの CSV の取込データ |
| 請求調整 | 現金・保留・追加請求など |
| 現金支払いマスタ | 毎月自動で現金扱いの人 |
| e集ちゃん取込 | e集ちゃん CSV の取込データ |
| 照合結果 | 照合の結果 |
| 操作履歴 | 操作ログ |
| 氏名確認済み | 氏名注意を確認済みにした ID |
| 設定 | マスタ参照設定 |

実行後、Google ドライブに「請求確定チェック」というスプレッドシートが増えています。  
中身を開くと、上記シートが作られていれば成功です。最初は見出し行だけでデータ行は空です。

---

### ステップ4: 設定が正しいか確認する（任意）

エディタで **`getSetupStatus`** を実行すると、登録済み ID を確認できます。

```json
{
  "version": "0.1.1",
  "billingSpreadsheetId": "（請求用SSのID）",
  "masterSpreadsheetId": "（マスタのID）"
}
```

どちらかが空文字の場合は、ステップ3が完了していません。

**手動確認（エディタの画面から）**

1. 左メニュー「プロジェクトの設定」（歯車アイコン）
2. 「スクリプト プロパティ」タブ
3. 次の2つがあること

| プロパティ名 | 値 |
|---|---|
| `BILLING_SPREADSHEET_ID` | 請求用スプレッドシートの ID |
| `MASTER_SPREADSHEET_ID` | 既存マスタの ID |

---

### ステップ5: Webアプリをデプロイする

`runInitialSetup` だけではブラウザからアプリは開けません。デプロイが必要です。

1. Apps Script エディタ右上の **「デプロイ」** → **「新しいデプロイ」**
2. 歯車アイコン（種類の選択）を押す
3. **「ウェブアプリ」** を選ぶ
4. 次のように設定する

| 項目 | 推奨設定 |
|---|---|
| 説明 | `請求確定チェック v0.1.1` など |
| 次のユーザーとして実行 | **自分（デプロイしたユーザー）** |
| アクセスできるユーザー | **自分のみ**（社内共有時は要相談） |

5. **「デプロイ」** を押す
6. 表示された **ウェブアプリ URL** をコピーしてブックマークする

**注意**

- コードを `clasp push` で更新したあとは、「デプロイの管理」→ 既存デプロイの「編集」→ **バージョンを「新バージョン」に変更」** してから再デプロイしないと、画面に古いコードが表示されることがあります。

---

### ステップ6: 動作確認

1. デプロイ URL をブラウザで開く
2. 左の「対象月」で今月（例: `2026-06`）を選ぶ
3. **「読み込み」** を押す
4. ダッシュボードにエラーが出ず、件数が 0 でも表示されれば起動 OK
5. 「ほのぼの取込」タブで CSV を1件試し取込

**うまくいっているサイン**

- 「スクリプト プロパティが設定されていません」エラーが出ない
- 請求用スプレッドシートの「請求月管理」に対象月の行が追加される
- ほのぼの CSV 取込後、ダッシュボードの件数・金額が変わる

---

### よくあるエラーと対処

| エラーメッセージ | 原因 | 対処 |
|---|---|---|
| `MASTER_SPREADSHEET_ID が Script Properties に設定されていません` | セットアップ未実行 | ステップ3を実行 |
| `既存マスタのスプレッドシートIDを引数に指定してください` | ID 未指定で実行 | ID を入れて再実行 |
| `基本情報マスタが見つかりません` | マスタ ID またはシート名が違う | ID を確認。設定シートの `BASIC_INFO_SHEET_NAME` を確認 |
| 権限の承認画面が出る | 初回実行時の正常動作 | 指示に従って許可 |
| Webアプリが真っ白 / 古い画面 | デプロイが古い | 新バージョンで再デプロイ |

---

### ステップ2以降を省略したい場合（手動設定）

`runInitialSetup` を使わず手動でも構いません。

1. Google ドライブでスプレッドシート「請求確定チェック」を新規作成
2. Script Properties に手動登録
   - `BILLING_SPREADSHEET_ID` = 手順1の ID
   - `MASTER_SPREADSHEET_ID` = 既存マスタの ID
3. エディタで **`setupApplicationSheets`** を実行してシート作成

ただし **`runInitialSetup` の方が簡単** なので、通常はそちらを推奨します。

## 管理用関数

| 関数 | 用途 |
|---|---|
| `runInitialSetup(masterSpreadsheetId)` | 初回のみ。請求SS作成とプロパティ設定 |
| `setupApplicationSheets()` | シート不足時の再作成 |
| `getSetupStatus()` | 設定済み ID の確認 |
